# План реализации V0

Инкрементально: каждый этап — отдельный работающий срез. Не забегать вперёд.
Оценки относительные (S/M); зависят прежде всего от готовности промпта.

| Этап | Что делаем | Результат / задачи | Оцен. |
|---|---|---|---|
| 0 | Каркас бота | Бот в ТГ: `/start`, приём текста/файла, ответ-заглушка. Доказываем, что труба работает. Регистрация в BotFather, токен в `.env`. | S |
| 1 | Вертикальный срез на тексте/`.txt` | Текст → ядро (промпт + Claude, структурный JSON) → исправленный текст файлом назад. Первый реально работающий бот. | M |
| 2 | Форматы файлов | `extract(file)` для всех форматов: плоские `.txt/.md/.csv` — чтением, `.docx/.odt/.rtf` — через pandoc. Детект пустого/нечитаемого файла. | S |
| 3 | Качество и формат вывода | Доводка промпта на эталонных текстах; словарь англицизмов как данные; саммари правок; формат выходного файла (`.txt`/`.md`/опц. `.docx`). | M |
| 4 | Устойчивость и деплой | Лимиты размера/длины, обработка ошибок, паттерн «Обрабатываю…», логирование, выкатка на PaaS/VPS, бот стабильно онлайн. | M |
| позже | Рост (вне V0) | ТОВ в промпт · Notion-адаптер с записью обратно · память (V2). Швы под всё это уже оставлены. | — |

## Definition of Done (приёмка V0)
- Бот принимает вставленный текст и файлы `.txt/.md/.csv/.docx/.odt/.rtf`, возвращает исправленный текст файлом + саммари.
- Орфография, пунктуация, грамматика и англицизмы обрабатываются на эталонном наборе реальных текстов так, что редактор принимает большинство правок.
- Неоправданные англицизмы заменены русскими аналогами прямо в тексте.
- Понятные сообщения на неподдерживаемый ввод.
- Один деплой; бот стабильно онлайн; словарь/правила правятся без изменения кода.

## Риски и митигации

| Риск | Митигация |
|---|---|
| Потеря вёрстки в `.docx/.odt/.rtf` | V0 заявляет работу с текстом, а не с вёрсткой; вывод — текст/markdown. Сохранение форматирования — отдельный кусок вне V0. |
| Повреждённый/нечитаемый файл | Обработка ошибок извлечения: понятное сообщение «не смог прочитать файл, пришлите в другом формате». |
| Длинные документы → лимит токенов / стоимость | Кап символов на запрос; при превышении — разбиение на части или отказ с пояснением. |
| Качество русской редактуры у модели | Итерация промпта на эталонном наборе; опц. детерм. спелл-чек как дополнение к слою корректуры. |
| Стоимость на объёме | Модель Sonnet (баланс); измерять токены; правила держать компактными в промпте. |

## Эталонный набор (для этапа 3)
Собрать 5–10 реальных текстов агентства с известными «правильными» правками и гонять на них
после каждого изменения промпта. Метрика: доля правок, принимаемых редактором без изменений.
