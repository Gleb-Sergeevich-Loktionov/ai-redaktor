# Архитектура V0

См. схему: `docs/diagram_v0.png`.

## Принцип: одно ядро + тонкий адаптер канала
Ядро редактуры не знает про Telegram. Telegram — лишь одна дверь к нему. Завтра Notion или
веб — это ещё один тонкий адаптер за тем же контрактом, ядро не меняется (Optimize for Change).
Это разделение — Separation of Concerns + DRY: логика редактуры пишется один раз.

## Поток (линейный конвейер)
```
Пользователь (Telegram)
  → Бот-адаптер  (① сразу «Обрабатываю…»)
    → Скачать файл + проверка типа/размера
      → Извлечение текста  (.txt/.md/.csv → чтение; .docx/.odt/.rtf → pandoc)
        → ЯДРО edit(text)  (промпт + словарь → один вызов Claude → JSON)
          → Сборка ответа  (файл с текстом + саммари)
            → ② Бот отправляет результат пользователю
```
Состояния между запросами нет: ни БД, ни очередей, ни Notion.

## Компоненты и ответственность

| Компонент | Ответственность / обоснование |
|---|---|
| Бот-адаптер | Принимает текст/файл, качает, валидирует, зовёт ядро, возвращает результат, ведёт UX (статусы, ошибки). **Тонкий** — логики редактуры здесь нет. |
| Извлечение текста | Узкий интерфейс `extract(file) -> str` (Law of Demeter). Плоские `.txt/.md/.csv` — чтением; `.docx/.odt/.rtf` — через pandoc. `.pdf` добавляется здесь же позже. |
| Ядро `edit(text)` | Сердце системы. Собирает промпт (правила + словарь) + один вызов Claude → структурный JSON. Пишется один раз, переиспользуется любым каналом. |
| Словарь + правила | `config/anglicisms.json` и `config/rules.md` — **данные**, правятся без кода, подмешиваются в промпт. Сюда же позже ляжет ТОВ. |
| Сборка ответа | JSON → выходной файл (исправленный текст) + текст саммари. Отделена от анализа, чтобы менять формат вывода независимо. |

## Контракт ядра (стабильный шов)
```
edit(text: str, rules: str, glossary: dict) -> {
  edited_text: str,                 # готовый исправленный текст
  summary: { fixes_total: int, anglicisms_replaced: [str], style_notes: [str] },
  comments: [ { type, fragment, change, reason } ]   # опц., для разбора
}
```
Канал может меняться — контракт нет. Подробности промпта и формата — `PROMPT.md`.

## Целевая раскладка `src/` (ориентир, не догма)
```
src/
  bot.py            # Telegram-адаптер: хэндлеры, скачивание файла, ответы, ошибки, паттерн «Обрабатываю…»
  extract.py        # extract(file) -> str: .txt/.md/.csv чтением; .docx/.odt/.rtf через pandoc
  core.py           # edit(text, rules, glossary): сборка промпта + вызов Claude + парсинг JSON
  prompts.py        # шаблон системной инструкции + сборка промпта (см. PROMPT.md)
  config_loader.py  # загрузка config/anglicisms.json и config/rules.md
  render.py         # JSON -> выходной файл + текст саммари
  settings.py       # чтение .env (токены, модель, лимиты)
```
Это **модули одного сервиса**, а не микросервисы. Если модуль не оправдан на текущем скоупе — не создавай его.

## Стек (boring, обоснован)

| Слой | Выбор | Почему |
|---|---|---|
| Язык | Python | Стандарт и для ТГ-ботов, и для LLM. Минимум кода. |
| Telegram | python-telegram-bot, long-polling | Конвенциональная библиотека. Long-polling — нет публичного HTTPS/SSL на старте. Webhook — при росте нагрузки. |
| Извлечение | pandoc + прямое чтение | Один зрелый конвертер на `.docx/.odt/.rtf`; плоские форматы читаются напрямую. PDF/OCR — позже. |
| Модель | Claude Sonnet (`claude-sonnet-4-6`) | Баланс качество/стоимость для редактуры. Haiku — если качество удержится; Opus — только если потребует качество. |
| Хостинг | один контейнер на PaaS / VPS | Без Kubernetes. Платим за один маленький сервис. |
| БД / очереди / Notion | нет | YAGNI на текущем скоупе. |

## Долгая обработка (единственный «асинхрон» V0)
LLM-вызов занимает секунды. Не блокируем обработчик: бот сразу отвечает «Обрабатываю…»,
затем шлёт результат отдельным сообщением (фоновая задача в процессе бота). Полноценная
очередь не нужна, пока объём мал — добавляем только при реальном росте нагрузки.

## Стоимость
Линейна по длине текста (токены вход + выход). Для предсказуемости — кап символов на запрос
(`MAX_CHARS` в `.env`). Актуальные цены модели сверять по докам Anthropic, в коде не зашивать.
