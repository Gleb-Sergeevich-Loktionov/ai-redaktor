# AI-редактор — Telegram-бот (V0)

> Этот файл Claude Code читает в начале каждой сессии. Держим его коротким и высокосигнальным.
> Подробности — в `docs/` (читай по необходимости, здесь не дублируем).

## Что это
- Telegram-бот. На вход: **вставленный текст или файл** `.txt .md .csv .docx .odt .rtf`.
- Что делает: исправляет **орфографию, пунктуацию, грамматику** и **заменяет неоправданные англицизмы** на русские аналоги. Лёгкая стилевая правка по общей норме.
- На выход: **файл с исправленным текстом + краткое саммари правок**.
- Технически: **один вызов Claude на текст**. Без состояния между запросами.

## Архитектура (одно ядро + тонкий адаптер)
- **Ядро** `edit(text, rules, glossary) -> {edited_text, summary, comments}` — промпт + один вызов Claude.
  Ядро **ничего не знает про Telegram**.
- **Telegram-адаптер** — тонкий: принять → извлечь текст → позвать ядро → вернуть результат.
- Новый канал (Notion, веб) в будущем = ещё один тонкий адаптер за тем же контрактом. Ядро не трогаем.
- Детали и контракт — `docs/ARCHITECTURE.md`. Промпт ядра — `docs/PROMPT.md`.

## Скоуп V0 — чего НЕ делаем (важно: не переусложнять)
- **НЕТ базы данных.** Состояния между запросами нет — хранить нечего.
- **НЕТ Notion-интеграции.** Это отдельный адаптер позже.
- **НЕТ памяти / накопления контекста / агентной оркестрации.** Поток фиксирован. Это V2.
- **НЕТ OCR и PDF** в V0 (добавляются позже через тот же интерфейс извлечения).
- **НЕ сохраняем исходную вёрстку** `.docx/.odt/.rtf` — извлекаем текст, отдаём текст/markdown.
- **НЕТ веб-UI, аккаунтов, ролей, биллинга, очередей.**
- Полная редактура по ТОВ/стратегии — **слот в промпте есть, но пустой** (ТЗ неполное; подключим, когда появятся документы).
- Полный список и обоснования — `docs/SPEC.md`.

## Стек (boring, обоснован)
- **Python.**
- **python-telegram-bot**, режим **long-polling** (не webhook — нет публичного HTTPS и SSL на старте).
- Извлечение текста: плоские `.txt/.md/.csv` — чтением; `.docx/.odt/.rtf` — через **pandoc**.
- **anthropic** SDK, модель **`claude-sonnet-4-6`** (баланс цена/качество). Структурный вывод (JSON).
- Хостинг: **один контейнер** на PaaS / небольшой VPS. **НЕ Kubernetes.**
- Сверять актуальные цены/лимиты модели по докам Anthropic; в код не зашивать.

## Конвенции
- Идентификаторы в коде — английские; комментарии/доки — русский ок.
- **Словарь англицизмов и правила стиля — это данные** (`config/`), не код. Правятся без изменения кода и подмешиваются в промпт.
- Секреты только из `.env` (см. `.env.example`). Никогда не коммитить реальные ключи.
- Один сервис, слои — это **модули**, а не микросервисы. Логику редактуры держим в ядре, не в адаптере.
- Перед добавлением библиотеки/абстракции/слоя — спроси: это правда нужно для текущего скоупа? Если нет — не добавляй (YAGNI).

## Команды
> Заполняются по мере реализации; держи в актуальном состоянии.
- Установка: `pip install -r requirements.txt` (нужен системный `pandoc`).
- Запуск бота: `python -m src.bot`  *(уточнить при реализации)*
- Тесты: `pytest`  *(когда появятся)*
