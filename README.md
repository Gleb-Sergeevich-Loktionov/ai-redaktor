# AI-редактор — Telegram-бот

Telegram-бот, который принимает текст или файл (`.txt`, `.md`, `.csv`, `.docx`, `.odt`, `.rtf`),
исправляет орфографию, пунктуацию, грамматику и заменяет неоправданные англицизмы на русские
аналоги, и возвращает исправленный текст файлом + краткое саммари правок.

## Документация
- `CLAUDE.md` — контекст и правила проекта (читает Claude Code в начале сессии).
- `docs/SPEC.md` — спецификация V0 (скоуп, не-цели, вход/выход, trade-offs, DoD).
- `docs/ARCHITECTURE.md` — архитектура, контракт ядра, целевая раскладка `src/`.
- `docs/PLAN.md` — план реализации по этапам, риски.
- `docs/PROMPT.md` — устройство промпта ядра и формат ответа.
- `docs/PRINCIPLES.md` — инженерные принципы проекта.
- `docs/diagram_v0.png` — схема потока.

## Данные (правятся без кода)
- `config/anglicisms.json` — словарь англицизмов (термин → замена).
- `config/rules.md` — правила стиля.

## Работа с Claude Code
Открой папку проекта и запусти `claude` — он автоматически прочитает `CLAUDE.md` и поймёт контекст.
Реализацию веди по этапам из `docs/PLAN.md`. Если правишь архитектуру или скоуп — обнови
соответствующий файл в `docs/`, чтобы контекст не расходился с кодом.

## Быстрый старт (после реализации)
1. `cp .env.example .env` и заполнить токены.
2. Установить системный `pandoc` (для `.docx/.odt/.rtf`).
3. `pip install -r requirements.txt`.
4. Запуск: `python -m src.bot` *(уточнить при реализации)*.

## Статус
V0 — в разработке. Без БД, без Notion, без памяти: см. не-цели в `docs/SPEC.md`.
